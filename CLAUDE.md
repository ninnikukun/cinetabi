# CLAUDE.md — シネたび (CINETABI)

このファイルは Claude Code がセッション開始時に読み込む、プロジェクトの前提メモです。

## プロジェクト概要
映画記録アプリ「シネたび」。観た映画を記録し、近くの映画館を探し、承認制でゆるく友だちと繋がれる。
- 技術構成：Vite + React（ほぼ単一ファイル `src/App.jsx`）+ Supabase（認証・DB・RLS）
- デプロイ：Vercel（GitHub `main` ブランチに push すると自動デプロイ）
- 公開URL：https://cinetabi.vercel.app
- リポジトリ：github.com/ninnikukun/cinetabi

## 開発者について
- 非エンジニア（ヤメピ）。ターミナル・Gitに不慣れなので、コマンドは1つずつ明確に案内する。
- Claude Codeのモデルは用途で使い分けている：
  - 普段の細かい修正・バグ修正 → Sonnet
  - DB設計・RLS・複数ファイルにまたがる大きめの実装 → Fable（`/model` で切替）
  - 設計相談・見た目のモック作りは、別のチャット（Claude.ai）で行っている

## アーキテクチャの要点
- 認証：匿名スタート（`signInAnonymously`）→ 後から Google / メールを `linkIdentity` / `updateUser` で連携
- 記録データ：`records` テーブル（Supabase）。RLSで「本人のみ」＋（ステージ2以降）「承認済みフォロワーも閲覧可」
- プロフィール：`profiles` テーブル。`public_id`（usr_xxxx）でユーザーを識別。表示名は `display_name`
- フォロー機能（ステージ2）：`follows` テーブル。片方向フォロー＋承認制（全員が鍵垢）。
  ID＋名前の完全一致でのみ申請可能（`request_follow` RPC でサーバー側強制）。拒否は行削除（通知なし）。
  **匿名ユーザーはフォロー機能を使用不可**（申請・承認どちらもRPC/RLSでブロック）。
- デザイン：ダーク×アンバー（#0c0d16 / #e8b04b）。フォントは Oswald / Zen Kaku Gothic New / Space Mono
- ホーム画面のルック：「NEGA縦フィルム」＝16:9のコマが縦に並ぶ。ポスターはぼかし背景＋中央表示。
  月が変わる位置に英語月名だけの継ぎ目（例：`JULY`）が入り、タップで月間リキャップが開く
- 記録詳細：Instagram風の縦フィード（記録→記録へ縦スクロール）。各記録内はポスター⇄おもいで写真を横スワイプ

## ローカル開発の注意点（ハマりやすいポイント）

### `.env` は必ずUTF-8で保存する
PowerShellの `echo "..." > .env` は**UTF-16(LE, BOM付き)**で保存してしまうことがある。
Viteの環境変数パーサーはUTF-8前提のため、UTF-16のままだと `VITE_SUPABASE_URL` 等が読めず、
`supabase` クライアントが `null` になり、**エラーも出さずに黙ってローカル保存モード（LocalApp）に
フォールバックする**。「記録一覧は表示されるのにログイン関連の機能が一切出ない」という時は、
まずこれを疑う。`.env` を作る・編集する時は、テキストエディタ（メモ帳等）で保存するか、
文字コードを明示できる方法で作成すること。

### ポートの重複に注意
`npm run dev` を複数のターミナル（自分の手元 + Claude Codeがバックグラウンドで立てたもの）で
同時に実行すると、Viteは自動で別ポート（5173→5174→…）にずれる。「コードを直したのに反映されない」
と思ったら、実際に見ているポート番号と、コードが動いているポート番号が一致しているか確認する。
作業が一区切りしたら、不要なバックグラウンドサーバーは停止する。

### Supabaseメール送信のレート制限
無料プランはメール送信数に1時間あたりの上限がある。マジックリンク／メール連携のテストを
短時間に何度も行うと `email rate limit exceeded` になる。テスト用アカウントは使い回すか、
複数アカウントでの動作確認はGoogle連携（別Googleアカウントがあれば）と組み合わせる。

### Windows特有のコマンド
- `cd ~\Desktop` 等のチルダ展開が効かないことがある → `cd C:\Users\<ユーザー名>\...` とフルパスで指定
- PowerShellでスクリプト実行がブロックされる場合 → `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`

## Git運用
- 個人開発、mainブランチに直接コミットでOK（PRなどの運用はしていない）
- コミット前に一度ビルド確認（`npm run build`）してからコミットする
- 大きめの変更（DBスキーマ変更など）の前に、区切りとして一度コミットしておくと安全
