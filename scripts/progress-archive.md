# 映画館データ全国展開 進捗記録（生ログアーカイブ）

> **これは全国展開作業（2026年、関東→中部→関西→その他の4ブロック）の詳細な作業ログです。**
> 現在のパイプラインの状態が一目でわかるサマリーは `scripts/progress.md` を参照してください。
> このファイルは「なぜこの館がこうなっているか」「あの時どう判断したか」を後から追いたい時の
> 一次資料として残しています。都道府県単位の処理順に時系列で記録されています。

`scripts/README.md` のパイプラインで、地方ブロック単位（関東→中部→関西→…）に処理した。
各ブロックの結果をここに記録し、ブロック終了時に1コミット＋Opusレビューを行った。

進行順: 関東 → 中部 → 関西 → その他

**関東ブロック完了後のルール変更（中部ブロックから適用）**:
- website優先順位: OSMのwebsiteタグ > wiki/Wikipediaの公式サイト > null（モールドメイン誤爆対策）
- 閉館判定は`[closed]`候補を機械的に採用せず、必ず営業中リストとの重複を目視確認
- 対象ページの洗い出しは`scripts/wiki-page-list.txt`のマスターリストを使う
- データファイル追加後はdevサーバー再起動が必要

**中部ブロック完了後のルール変更（関西ブロックから適用）**:
- websiteルールを厳格化: OSMタグも無く、wiki上の情報がモールドメイン（施設全体のサイトで
  個別の上映スケジュールに辿り着けないもの）しか無い場合はnullにする
- null化した館・未取得の館は`scripts/website-todo.json`にリスト化（自動処理せず後で個別調査）
- `check-osm-closed.js`にOSM実座標との距離比較チェックを追加（座標乖離の早期発見用）
- 「開館前」表記の検出ロジックを継続適用し、ブロック完了後に過去ブロックも再チェック

### ブロック完了レビュー（Opus）を受けての追加対応

Opusレビューで指摘された3点のうち、影響が小さく即時対応できた1点をここで反映：

1. **【対応済み】関東ブロックの「開館前」見落としチェック**: 「開館前」検出ロジックは中部ブロック途中
   （TOHOシネマズ名古屋栄の件）で追加したため、それより前に処理した関東ブロックのスクレイプ結果は
   このチェックを経ていなかった。保存済みのスクレイプ生データを遡って確認したところ、
   東京都（残り21区+多摩4市）分に1件該当：**イオンシネマ八王子滝山**（wiki上は「開館前」表記のまま
   だったが、実際は2026年6月26日に開業済み・WebSearchで確認）。`tokyo-rest.json`に追加
   （lat/lon: 35.6844049 / 139.3415418、website: https://www.aeoncinema.com/cinema/hachiojitakiyama/、
   precision: poi）。他の関東ブロック県（神奈川・埼玉・千葉・茨城・栃木・群馬・渋谷区・新宿区分）を
   含む全スクレイプ生データを再チェックしたが、該当は上記1件のみ。build再実行・JSON検証済み。
   → 全国合計をcurated 213件に更新（上記反映後）。

2. **【要ユーザー判断・未対応】website優先ルールの実装とREADME記載の不一致**: `scripts/README.md`には
   「モールドメインらしきURLしかなくOSMにもタグが無い場合は無理に埋めずnullのままにする」と書いてあるが、
   実際の中部ブロックの処理では、この「モールドメイン疑いなら明示的にnullにする」判定は行っておらず、
   OSM > wiki/Wikipedia > null の機械的な優先順位のみを適用していた。結果として、OSM側にタグが無い
   一部の館ではモール/商業施設ドメインのURLがそのまま「公式サイト」として残っている
   （例: TOHOシネマズ浜松→zazacity.jp、MOVIX清水→dream-plaza.co.jp、TOHOシネマズ岐阜→colorfultown.jp、
   イオンシネマ系の一部→aeon.jp/sc/等の店舗ページ）。関東ブロックでも同種の「後回し」候補が約15件残っている
   （上記セクション参照）。どちらの運用にするかはユーザー判断が必要なため、今回は変更せず次回報告で確認する。

3. **【関西ブロック以降の改善案・未実装】curated座標とOSM座標の突き合わせチェック**: 現状の座標QAは
   「同一sourcePage内で15km以上離れた2件がないか」という粗いチェックのみで、浜松市のケース（同一市内
   だが30km近くずれていた）のような中距離の誤りは拾えるが、単館の市区町村や、同じ方向に2件ともずれて
